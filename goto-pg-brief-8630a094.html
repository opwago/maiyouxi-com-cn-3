关于 goto-pg.com

面向实用场景的 PG模拟器 · 从原理到交互的轻量化工具

PG模拟器 并非一个娱乐化模拟平台，而是一套面向数据库开发与教学场景的交互式环境。 本站以 PostgreSQL 行为为参照，提供可在线执行的 SQL 模拟层，帮助使用者快速验证查询逻辑、 理解隔离级别与执行计划的基础概念，而无需在本地搭建完整数据库实例。

设计特点与实用边界

⚙️ 行为模拟

支持常见 DML、事务控制、窗口函数及部分系统函数。模拟器尽力贴近真实 PG 语法容错与类型系统，但仍存在差异。

🧪 轻量交互

无需注册，打开即用。适用于快速原型、教学演示、或面试前语法回忆。不存储用户数据，会话状态临时保留。

📘 文档与示例

内置常用示例数据集与查询模板，降低上手成本。也适合作为 PostgreSQL 入门时的辅助练习环境。

🔒 边界说明

模拟层不连接真实数据库，不支持持久化、存储过程、复杂扩展。高并发场景下响应可能变慢，建议用于探索性查询。

* 模拟器并非真实 PG 实例，结果可能因版本或实现细节产生偏差。关键业务验证请以官方文档为准。

使用场景与可信说明

本站由独立开发者维护，并非 PostgreSQL 官方工具。代码开源，无第三方跟踪脚本，不收集个人标识信息。 建议在以下场景中使用 PG模拟器：

  • 快速测试 SQL 片段的语法与语义逻辑（如 JOIN、子查询、CTE）。
  • 教学环境中演示基础的查询计划与索引效果（模拟层提供简化 explain）。
  • 在无法安装本地数据库的设备上（如平板、受限网络）进行轻量练习。

需要提醒的是，模拟层对某些高级特性（如递归 CTE 深度、锁机制、GIS 函数）仅作有限支持。 若需要进行性能测试或生产环境验证，请直接使用本地 PostgreSQL 实例。站内不提供任何形式的“破解”“注入”或“绕过”功能。

常见疑问摘要

模拟器支持所有 PostgreSQL 数据类型吗？
支持大部分常用类型（数值、文本、时间、JSON、数组等）。部分自定义类型与域类型暂不包含。
我的查询会被记录或分享吗？
不会。所有查询在会话结束后清除。站点不设日志后端，亦不存储任何可回溯的查询文本。
和官方 psql 相比有哪些主要限制？
无连接池、无角色权限校验、无 pg_stat 统计信息、不保留 MVCC 可见性细节。请将其视为“语法兼容的沙箱”。
有没有离线版本或 API？
目前仅提供在线页面方式。项目暂不提供自部署镜像或 REST API。开发者可根据开源仓库自行构建。

如果你正在寻找一个轻量、专注的 SQL 实验环境，不妨试试 PG模拟器。它不替代真实数据库，却能在很多场景下节省你搭建环境的时间。
